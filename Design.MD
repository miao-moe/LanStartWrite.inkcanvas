# LanStartWrite 设计基线（Jalium.UI + Win11 Fluent）

本文件用于约束项目 UI 设计与实现，避免设置页、批注栏出现风格漂移和结构不稳定。

## 1. 窗口分层原则（必须遵守）

- 批注栏窗口：自绘浮动窗，透明宿主窗口，仅渲染工具栏实体。
- 设置窗口：标准系统窗口，使用正常标题栏与常规窗口行为。
- 两者禁止混用同一窗口策略（例如设置页禁止使用透明浮窗语义）。

## 2. Jalium.UI 实现原则

- 使用 `.jalxaml + code-behind`，避免在运行时拼装复杂视图树。
- 保持 `x:Name` 明确，交互控件在 code-behind 中强类型转换后使用。
- 颜色优先 `ThemeResource`，仅在关键可读性场景使用显式色值。
- 图标统一使用 `SymbolIcon + Symbol`，并在代码中集中设置画刷。
- 布局优先 `Grid`（结构）+ `StackPanel`（流式），减少多层空 `Border` 套娃。

## 3. Win11 Fluent 对齐规则

参考 Microsoft Learn（NavigationView / Settings / Spacing / Typography）提炼：

- 设置页必须有清晰导航骨架：左侧导航（可折叠）+ 右侧内容区。
- 左侧应提供汉堡入口，支持窄窗时紧凑导航。
- 内容区使用“分组标题 + 卡片行”，行内采用“左文本右控件”。
- 文本层级：页面标题 > 分组标题 > 设置项标题 > 辅助说明。
- 建议间距节奏：8 / 12 / 16 / 24 像素梯度。
- 设置项描述用简短句，避免长段解释堆叠。

## 4. 本项目设置页结构规范

- 导航项固定四组：外观、墨迹、窗口与交互、关于。
- 每个导航组对应一个内容面板，切换时仅显示一个面板。
- 未实现功能必须标记“预留”并禁用控件，不得伪装可用。
- 与批注行为强耦合的选项（例如置顶策略）要与当前行为一致。

## 5. 交互稳定性检查清单

- 打开/关闭设置窗口不会影响批注栏状态机。
- 切换导航不会抛异常、不会残留多面板重叠。
- 汉堡折叠后图标可见，展开后标签可见。
- 缩放窗口时，左侧导航和右侧内容不重叠、不裁切主控件。
- 所有符号图标与文本垂直居中。

## 7. Fluent 控件语义选择

- 二元开关（开/关）优先使用“开关/Toggle”形态，而不是复选框（CheckBox）。
- 多选互斥（从多个选项里选一个）优先使用下拉框（`ComboBox`）。
- 真多选才使用多选控件（如列表的多选模式），并确保视觉上不会堆叠出大量勾选框。

## 6. 后续演进建议

- 若 Jalium.UI 的 `NavigationView` 在当前版本稳定可用，优先迁移到 `NavigationView` 统一导航范式。
- 将设置项绑定到配置模型（持久化），避免 UI 仅做展示。
- 增加深色主题策略后，再放开“跟随系统主题”开关。
